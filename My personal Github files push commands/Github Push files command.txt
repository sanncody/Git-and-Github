1. git init (check default branch is main or master)

If it is master by default, then we can that initDefaultBranch to main using:

git config --global init.defaultBranch main


2. git status

3. git add

4. git commit -m "";

5. git remote add origin <Remote URL>

6. git pull origin main --allow-unrelated-histories

7. git push -u origin main


NOte: When we have to update our recent commit after it got pushed to main, then we can use:

 - git commit --amend -m "Your updated commit message"
 - git pull origin main 
 - git push origin main 



Note: Aapke local mein aapne galti se current branch pe kaam shuru kar diya tha, jisme pehle se kuch specific code tha jo us branch ke liye tha. Ab aapne apna next task bhi same branch pe shuru kar diya aur naya branch create karna bhool gaye.

Ab aapko apna non-committed work is current branch se nayi branch pe transfer karna hai. Yeh karne ke liye, yeh steps follow karein:

1. Apne current changes ko stash karo (jo abhi commit nahi kiye gaye hain):
 	
	git stash

2. Nayi branch create karo jo aapko apne next task ke liye chahiye:

	git checkout -b <New-branch-name>

3.*** Stash ko nayi branch pe apply karo: Isse aapke stash wale changes nayi branch pe apply ho jayenge.

	git stash pop

4. git status chalao


Ab aapke non-committed work ko nayi branch pe transfer kar diya gaya hai.

Agar aap purani branch se wo extra work hata bhi chahte hain, toh:

1. Purani branch pe wapas jao

	git checkout old-branch-name

2. Purani branch ke changes reset karo : Isse purani branch ke sab uncommitted changes hata diye jayenge.

	git reset --hard
